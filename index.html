VisitorLogic

A lightweight runtime that adapts website copy based on visitor context — without cookies, fingerprinting, or AI APIs.

Get Started

What VisitorLogic does

Most websites show the same message to every visitor. VisitorLogic allows your site to respond differently based on page type, audience, industry, region, and referrer — all resolved locally in the browser.

Zero tracking

No cookies. No user identity. No fingerprinting. Everything runs client-side.

Deterministic logic

No AI hallucinations. Every headline is intentional and reviewable.

Fast by default

No async blocking. No backend calls. No layout shift.

How it works

VisitorLogic resolves a segment key at runtime, then injects copy into your page using safe data attributes. 

page_type | audience | industry | region | referrer
homepage  | teams    | fintech  | africa | search

Installation

Drop this into your site — that’s it.

<!-- VisitorLogic Config -->
<script>
  window.VisitorLogic_CONFIG = {
    industry: "agriculture",
    audience: "teams",
    page_type: "homepage"
  };
</script>

<!-- VisitorLogic Runtime -->
<script src="https://cdn.jsdelivr.net/gh/yourname/visitorlogic/dist/visitorlogic.v1.js" defer></script>

Segment data

Segments live in a versioned JSON file. No data is hidden. Everything is inspectable. 

{
  "homepage|teams|agriculture|africa|direct": {
    "page_type": "homepage",
    "audience": "teams",
    "industry": "agriculture",
    "region": "africa",
    "referrer_type": "direct",
    "headline": "Built for growing agricultural teams",
    "subheadline": "Tools designed for real-world operations.",
    "cta": "Learn more"
  }
}

Wildcard resolution

If an exact match is not found, VisitorLogic gracefully falls back using wildcards — never breaking your page. 

homepage|teams|agriculture|africa|search
homepage|teams|agriculture|africa|*
homepage|*|agriculture|africa|*
homepage|*|*|global|*
default

❌ Not AI-heavy

No token costs. No LLM latency.

❌ Not fragile

No runtime errors if a segment is missing.
